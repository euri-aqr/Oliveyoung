Career Summary

5년 차 PM. 그 이전 10년의 콘텐츠 도메인 경력. 데이터 매거진과 큐레이션 서비스를 0에서 1로 만들었고, 이어서 글로벌 크리에이터 어필리에이트 플랫폼에 LLM 기반 AI 어시스턴트를 PM으로 빌딩·탑재했습니다. PM 이전에는 출판사에서 정기간행물 3종을 1인 풀사이클로 운영했고, 국제 콘텐츠 행사의 PR·마케팅을 담당했습니다.

"콘텐츠 → 사용자 신호 → 다음 의사결정"의 루프를 설계하는 일. 사내 데이터 매거진에서는 인터뷰 결과를 누적해 다음 상품 기획에 반영하는 사이클을 만들었고, AI 어시스턴트에서는 15주 분량의 대화 로그를 분석해 응답 정확도를 65%에서 88%로 끌어올렸습니다. 흩어진 정보를 사용자의 언어로 옮기고, 운영에서 나오는 데이터로 다시 정보 구조를 다듬는 일을 같은 방식으로 해 왔습니다.

발견탭에서 풀고 싶은 일은 매거진·셔터·리뷰·라이브가 만들어내는 콘텐츠 참여 신호를 PDP·구매·큐레이터 전환으로 잇는 데이터 루프 설계입니다. 콘텐츠 감각·데이터 분석·서비스 기획을 한 PM이 같이 쥐어야 만들 수 있는 구조라고 생각합니다.

Problem & Goal

구성원들은 "데이터 드리븐"의 필요성을 느꼈지만, 기본 용어부터 어려웠고 분석 보고서는 여러 공간에 흩어져 있었습니다. 대시보드나 교육 이전에 "읽고 싶은 콘텐츠 형태로 데이터 접근성을 높이는 일"이 먼저라고 판단했습니다. 분산된 데이터 서비스를 하나의 플랫폼으로 통합하는 0→1 프로젝트로 출발했습니다.

Action
  • 데이터 매거진 풀사이클 운영: 콘셉트, 섹션 구성, 발행 주기, 운영 체계까지 하나의 서비스 시스템으로 구조화. 5개 섹션(용어 사전, 도서 요약, 보고서 큐레이션, 협업 사례 인터뷰, 협업 가이드)을 매월 발행.
  • 데이터 큐레이션 서비스 화면 단위 기획: 사내 공지·런칭·타운미팅에서 키워드를 추출하고 관련 보고서를 테마 큐레이션으로 묶는 프로세스 설계. 메인 화면(추천 검색어, 추천 태그, 월 1회 테마 큐레이션, 정기·팀별 보고서 분류)과 검색 결과 화면(태그 필터, 카드형 결과)을 화면 단위까지 상세 설계.
  • 데이터 용어사전 기획: 1,000+ 데이터 용어를 카테고리·태그·관계로 구조화한 정보 아키텍처 설계.
  • 자연어 쿼리 챗봇 기획: 데이터 분석을 자연어로 수행 가능한 챗봇. 이후 AI 어시스턴트 프로젝트의 사전 경험이 됨.
  • BI 대시보드 베타 기획: 게임 KPI 시각화 대시보드. 사용자 요구사항 → 기능 범위·우선순위, 지표 구조·문서화 체계.
Result & Learning

매거진은 발행되어 구성원의 긍정적 피드백("친절하고 친근한 어투로 눈높이에 맞는 아티클을 제공해 줘서 좋았다")을 얻었으나, 조직 개편과 리소스 부족으로 프로젝트가 중단됐습니다. 편집위원회 3인 + 편집부 3인 체계를 설계해 두었지만 실제로는 그만큼의 인력이 필요한 일을 혼자 떠안게 되었던 것이 가장 큰 이유였습니다. 콘텐츠 품질을 지키는 것과 지속 가능한 운영 구조를 만드는 것은 다른 문제라는 점을 이때 배웠고, 이후 다음 프로젝트에서는 처음부터 팀 구조와 릴리즈 리듬을 함께 설계했습니다.

Discovery

운영 데이터와 CS 흐름을 보며 가입 자격·코드 발급·수익 정산 같은 어필리에이트 운영 영역에서 반복 문의가 축적되는 지점을 발견했습니다. 위에서 지시받은 과제가 아니라, 플랫폼 전체의 병목을 보며 직접 문제를 정의하고 제안한 프로젝트입니다.

탑재 대상은 자사의 글로벌 크리에이터 어필리에이트 플랫폼이었습니다. 콘텐츠 크리에이터·스트리머·아티스트가 고유 코드를 발급받고, 팔로워가 해당 코드로 자사 스토어에서 구매 시 크리에이터가 커미션을 분배받는 구조입니다. 여러 자사 게임 IP의 글로벌 마케팅을 크리에이터 네트워크 기반으로 전개하는 플랫폼입니다.

Strategy & Stakeholder Management

상위 기획부터 피벗과 솔루션 선정까지, 매 단계에서 근거를 모으고 이해관계자와 합의를 만들어 갔습니다.

  • 피칭으로 다부서 설득: 채택 프레임워크 → AI 어시스턴트 정체성 → 페르소나 → 경쟁 솔루션 → 사용자·시스템 플로우 → KPI 정의 순서로 상위 기획을 구조화. 관련 8개 이상 부서의 stakeholder를 설득해 프로젝트 합의를 끌어냈습니다.
  • 피벗 판단·합의: 초기 내부 사용자 플로우가 무산되자, 대외 사용자 문제로 초점을 재정의. 데이터·운영 흐름의 근거를 모아 새 로드맵을 합의하고 SaaS 솔루션을 선정.
  • 솔루션 선정 기준: 사용 사례, 어드민·분석·API 기능, 빠른 통합 가능성, 개발 리소스 부족 상황의 지원 가능성을 근거로 LLM 기반 SaaS를 채택.
Affiliate Domain Learning

크리에이터 어시스턴트를 만들기 위해 이 플랫폼의 어필리에이트 구조 전반을 학습하고 페르소나·지식 베이스에 반영했습니다.

  • 크리에이터 등급: 등급별 운영 정책 차별화.
  • 가입·코드·정산: 소셜 채널 연동, 고유 코드 발급, 글로벌 정산 흐름.
  • 캠페인 운영: 금전적·비금전적 캠페인 운영, 게임별 캠페인 차별화.
  • 글로벌 다국가 운영: 다국어 응대, 제재국 제외 글로벌.

크리에이터의 가장 큰 반복 문의가 가입·자격·코드·정산이었기 때문에, AI 어시스턴트의 지식 베이스는 이 어필리에이트 구조를 정확히 반영해야 했습니다. 이 도메인 지식은 콘텐츠 커머스의 큐레이터·어필리에이트 운영 흐름과 구조적으로 같은 결입니다.

Data Loop & Quality Operation

사용자 문의 로그를 그냥 쌓아두지 않고, 매주 분석해 문서 구조를 재편하고 다음 주의 응답 품질을 끌어올리는 사이클로 운영했습니다.

  • 지표 정의·운영: 응답 정확도, 해결률, Dead Ends 비율, 응답 일관성을 PM이 직접 정의.
  • 로그 기반 패턴 도출: 15주 분량의 대화 로그에서 미응답·오답 패턴 분석 → 문서 구조 최적화.
  • 개선 방식: RAG + 핵심 FAQ 사전 정의 병행, 문서 구조 재정렬, 커스텀 위젯, 세그먼트별 플로우.
  • 세그먼트별 개인화: 비로그인·일반·파트너 세그먼트에 따라 대화 흐름과 응답이 분기되는 구조 기획.
Execution & Team Building

초기 2인에서 FE·BE·디자인·QA·DE·DA가 참여하는 7인 협업 구조로 확장했습니다. 도구를 선택하는 것보다 같은 시간에 같은 의사결정이 일어나는 릴리즈 리듬을 만드는 일이 더 어려웠습니다. 요구사항·스펙·QA 이슈·회고를 Jira와 Confluence에 남기는 방식을 정착시켰고, 현재 운영 버전 v0.5에 이르기까지 정기 릴리즈 사이클을 운영하고 있습니다.
