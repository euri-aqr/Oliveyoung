Problem

"데이터 드리븐 조직"은
선언만으로 작동하지 않았습니다

구성원들은 데이터를 알아야 한다는 필요성은 느꼈지만, 기본 용어부터 어려워했고 분석 보고서는 여러 공간에 흩어져 있었습니다. 대시보드나 교육 이전에 읽고 싶은 콘텐츠 형태로 데이터 접근성을 높이는 것이 먼저라고 판단했습니다.

Action 1 · Magazine

매거진을 제품처럼 설계했습니다

콘셉트, 섹션 구성, 운영 프로세스, 발행 주기를 하나의 서비스 시스템으로 구조화했습니다.

용어 사전매월 10개 내외의 데이터 기본 용어를 눈높이에 맞게 설명
도서 요약추천 도서를 에디터가 읽고 요약·리뷰하는 "부탁하면 읽어드립니다" 콘셉트
보고서 큐레이션시의성 있는 이슈와 분석 보고서를 연결해 큐레이션
협업 사례 인터뷰부서 간 우수 협업 사례와 데이터 활용 사례 인터뷰
협업 가이드데이터 조직과의 협업 가이드를 콘텐츠로 재편집
Action 2 · Curation IA

보고서를 "찾을 수 있는 서비스"로 바꿨습니다

사내 공지·런칭·타운미팅에서 키워드를 추출하고, 관련 분석 보고서를 찾아 테마 큐레이션으로 묶는 프로세스를 설계했습니다. 메인 화면에는 추천 검색어, 추천 태그, 월 1회 테마 큐레이션, 정기·팀별 보고서 분류단을 배치했고, 검색 결과 화면은 태그 필터와 카드형 결과로 구성했습니다.

사내 이슈→ 키워드 추출→ 보고서 탐색→ 테마 선정→ 추천 아티클
Result & Learning

좋은 콘텐츠만으로는 부족했습니다

"친절하고 친근한 어투로 눈높이에 맞는 아티클을 제공해 줘서 좋았다. 데이터 관련 소통에서 느끼던 답답함이 일정 부분 해소되었다." — 구성원 피드백

매거진은 발행되어 구성원 반응을 얻었지만, 조직 개편과 리소스 부족으로 중단되었습니다. 편집위원회 3인 + 편집부 3인 체계를 설계해 두었지만 실제로는 그만큼의 인력이 필요한 일을 혼자 떠안게 되었던 것이 가장 큰 이유였습니다. 콘텐츠 품질을 지키는 것과 지속 가능한 운영 구조를 만드는 것은 다른 문제라는 것을 이때 배웠고, 이후 다음 프로젝트에서는 처음부터 팀 구조와 릴리즈 리듬을 함께 설계했습니다.

Discovery

크리에이터에게는
안내자가 필요했습니다

탑재 대상은 자사의 글로벌 크리에이터 어필리에이트 플랫폼이었습니다. 콘텐츠 크리에이터·스트리머·아티스트가 고유 코드를 발급받고, 팔로워가 해당 코드로 자사 스토어에서 구매 시 크리에이터가 커미션을 분배받는 구조입니다. 여러 자사 게임 IP의 글로벌 마케팅을 크리에이터 네트워크 기반으로 전개하는 플랫폼입니다.

운영 데이터와 CS 흐름을 보며 가입 자격·코드 발급·수익 정산 같은 어필리에이트 운영 영역에서 반복 문의가 축적되는 지점을 발견했습니다. 위에서 지시받은 과제가 아니라, 플랫폼 전체의 병목을 보며 직접 문제를 정의하고 제안한 프로젝트입니다.

Affiliate Context

어필리에이트 구조를
서비스 내부에서 학습했습니다

크리에이터 어시스턴트를 만들기 위해 이 플랫폼의 어필리에이트 구조 전반을 학습하고 페르소나·지식 베이스에 반영했습니다.

  • 크리에이터 등급: 수동 승인 등급과 자동 승인 등급 2-tier 구조. 등급별 운영 정책 차별화
  • 가입·코드·정산: 소셜 채널 연동, 고유 코드 발급, 글로벌 정산 흐름
  • 캠페인 운영: 금전적·비금전적 캠페인 운영, 게임별 캠페인 차별화
  • 글로벌 다국가 운영: 다국어 응대, 제재국 제외 글로벌

크리에이터의 가장 큰 반복 문의가 가입·자격·코드·정산이었기 때문에, AI 어시스턴트의 지식 베이스는 이 어필리에이트 구조를 정확히 반영해야 했습니다. 이 도메인 지식은 올리브영 쇼핑 큐레이터(어필리에이트) 운영 흐름과 구조적으로 같은 결입니다.

Strategy

Execution

도구보다 중요한 것은
반복 가능한 리듬이었습니다

초기 2인에서 FE·BE·디자인·QA·DA가 참여하는 7인 협업 구조로 확장했고, 현재 운영 버전 v0.5에 이르기까지 정기 릴리즈 사이클을 정착시켰습니다. 요구사항·스펙·QA 이슈·회고를 Jira와 Confluence에 남기며 같은 시간에 같은 의사결정이 일어나는 리듬을 만드는 일이 도구를 선택하는 일보다 더 어려웠습니다. 지금은 단순 FAQ 챗봇을 넘어 사용자 맥락을 이해하고 선제적으로 돕는 Proactive AI로의 전환을 추진하고 있습니다.

어필리에이트 도메인이 플랫폼을 통해 크리에이터 가입·자격·코드·수익 분배·캠페인 운영 등 어필리에이트 구조를 서비스 내부에서 인접 학습. 쇼핑 큐레이터 운영에 직접 적용 가능
지표 운영콘텐츠 CTR·PDP 진입률·저장·공유율·큐레이터 전환율 같은 지표도 동일하게 정의·관찰·개선 가능
로그 기반 분석대화 로그의 미응답·오답 패턴 분석을 콘텐츠 소비 로그의 이탈·클릭·탐색 패턴 분석으로 전환 가능
프로젝트 관리실험 가설·개발 범위·운영 정책·QA·회고를 하나의 릴리즈 리듬으로 묶는 역량
변화 대응초기 가설 무산 시 근거 기반 합의를 이끌고 새 로드맵 수립
05 · Background

콘텐츠 생산 현장에서
구조를 만들어 온 시간

매거진을 만들 수 있었던 것도, AI 챗봇의 페르소나를 설계할 수 있었던 것도, 그 이전에 콘텐츠 생산과 소비의 현장에서 구조를 만들어 온 시간이 있었기 때문입니다.

Editorial
정기간행물 3종 1인 풀사이클 운영
금성출판사 · 2014–2017

어린이·교육 정기간행물 3종을 혼자 기획·취재·기사 작성·발행했습니다. 매월 누적되는 인터뷰 결과를 부서의 다른 상품 기획에 반영하는 루프를 만들어, 단순 발행을 사용자 인사이트가 다음 상품으로 흐르는 데이터 사이클로 운영했습니다.

"감"이 아니라 "체계"로 콘텐츠를 만드는 습관은 월간지 3종을 1인 운영하며 만들어졌습니다.

06 · OliveYoung Discovery Loop

발견탭의 다음 기회는
콘텐츠 참여 신호를 커머스 루프로 연결하는 일이라고 봅니다

외부 공개 자료와 사용자 관찰을 바탕으로 정리한 가설들입니다. 실제 적용 전에는 내부 지표·운영 정책과 맞춰 보아야 합니다.

Ⅰ 사용자 신호를 다음 액션으로 옮긴다면
신호해석다음 액션 후보
상품 태그 클릭 콘텐츠에서 상품 관심으로 넘어간 순간 동일 상품의 리뷰·셔터·라이브 클립 모아보기
저장·공유 구매 전 탐색 또는 타인 추천 의도 큐레이터 링크 생성 CTA, 추천 콘텐츠 템플릿 제공
라이브 클립 시청 사용법·루틴·전문가 신뢰에 반응하는 단계 PDP 내 관련 루틴 콘텐츠 재노출
리뷰 탐색 검증·비교 단계 리뷰 요약, 피부고민·카테고리 태그 정렬
Ⅱ 발견 루프 단계별 가설
A Reach
첫 방문자에게는 "인기 피드"보다 "관심사 기반 입문 피드"가 더 작동할 것
Hypothesis 피부 고민·관심 카테고리 기반 첫 5개 카드가 상품 태그 CTR을 높일 것
Experiment 온보딩 2~3문항으로 관심사 추정 후 개인화 피드와 인기 피드 비교
Metrics 상품 태그 CTR, 2-depth 진입률, 첫 세션 체류시간
Guardrail 온보딩 이탈률, 반복 노출 피로도, 신규 유저 전환 저하
B Explore
상품 태그 클릭 이후에는 "이탈"이 아니라 "콘텐츠 이어보기"가 필요할 것
Hypothesis PDP에서 같은 상품의 셔터·리뷰·라이브 클립을 모아주면 탐색 지속률이 높아질 것
Experiment "이 상품 콘텐츠 모아보기" 모듈 A/B 테스트
Metrics PDP 체류, 장바구니 전환, 리뷰·콘텐츠 재진입률
Guardrail 관련성 낮은 콘텐츠 노출, PDP 핵심 정보 가림, 페이지 로딩 지연
C Convert
큐레이터 전환은 "수익 안내"보다 "콘텐츠 템플릿"이 더 효과적일 것
Hypothesis 사내 크리에이터 플랫폼에서 신규 크리에이터의 첫 콘텐츠 진입 패턴을 보면, 코드 발급 후 첫 콘텐츠 제작·공유까지의 시간이 가장 큰 병목이었습니다. 쇼핑 큐레이터도 수익률보다 "무엇을 어떻게 써야 하는지"에서 막힐 가능성이 높습니다.
Experiment 카테고리별 추천 문안·해시태그·이미지 구조 템플릿 제공군과 기존군 비교
Metrics 링크 생성률, 첫 공유율, 링크 클릭률, 구매 전환율
Guardrail 과장 표현, 저품질 추천, 브랜드 가이드 위반
D Retain
라이브는 방송 중에만이 아니라 "방송 후 재순환"되어야 더 오래 작동할 것
Hypothesis 라이브 핵심 장면을 짧은 클립으로 재가공해 셔터·PDP에 재배치하면 비시청자에게도 신뢰 맥락을 전달할 것
Experiment 방송 종료 후 24시간 내 PDP·셔터 피드에 핵심 클립 노출
Metrics 클립 시청 완료율, PDP 전환, 재방문율, 브랜드 팔로우
Guardrail 실시간 매력 저하, 라이브 본방 시청률 감소, 클립 품질 편차
07 · 90-Day Plan

처음 90일은
이해 → 실험 → 운영화로 움직이고 싶습니다

빠르게 상황을 파악하고, 작게 검증한 뒤, 성과가 나온 실험을 운영 정책과 템플릿으로 고정하는 방식으로 일하고 싶습니다.

30D
Understand
  • 발견탭·셔터·매거진·리뷰·라이브·큐레이터 현황 지표 파악
  • 진입·체류·클릭·저장·구매·공유 퍼널 재정의
  • 운영 정책·승인 프로세스·콘텐츠 생산 리드타임 파악
60D
Experiment
  • 관심사 피드·PDP 콘텐츠 모아보기·큐레이터 템플릿 중 1~2개 실험 설계
  • 성공 기준과 가드레일 지표 선합의
  • Jira·Confluence 기반 실험 문서 템플릿 정리
90D
Systemize
  • 성과가 확인된 실험을 운영 정책·대시보드·회고 리듬으로 고정
  • 크리에이터·큐레이터 운영 부담을 줄이는 도구화 과제 도출
  • 다음 분기 Discovery Loop 로드맵 제안
핵심 KPI 후보
콘텐츠 노출 대비 클릭률, 시청·읽기 완료율, 저장·공유율, 댓글·반응률
커머스 상품 상세 진입률, 장바구니·구매 전환율, 큐레이터 링크 생성·클릭·구매율
운영 콘텐츠 생산 리드타임, 큐레이션 승인 SLA, 재사용 가능한 실험 템플릿 수
